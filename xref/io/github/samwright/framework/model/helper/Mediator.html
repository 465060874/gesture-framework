9   /**
10   * Mediator objects contain data that was produced from a {@link Processor} object, and can
11   * be fed into another {@code Processor} object.
12   * <p/>
13   * Each {@code Mediator} has a link to the {@code Mediator} object which was fed into the
14   * {@code Processor} that created it (ie. the 'previous' {@code Mediator}).  The first
15   * {@code Mediator} in this linked list is an 'empty' {@code Mediator} (created using
16   * {@code Mediator.createEmpty()}) where both the data contained in it and the previous
17   * {@code Mediator} are {@code null}, and the {@link History} is {@code History.getEpoch()}.
18   * <p/>
19   * Each {@code Mediator} also has a {@code History} object which represents the list of
20   * {@code Processor} objects which created it.  Two {@code Mediator} objects which were created by
21   * the exact same procession of {@code Processor} objects will share the same {@code History}
22   * object (ie. {@code mediator1.getHistory() == mediator2.getHistory())} regardless of whether the
23   * {@code Mediator} objects' data are the same or not.
24   */
25  public abstract class Mediator {

29       * {@code Mediator} objects (which actually contain data) are created.  Its 'data' and
30       * 'previous' variables are null, and its {@link History} object is set at the epoch (ie.
31       * {@code History.getEpoch()}).
32       *
33       * @return an empty {@code Mediator}, from which {@code Mediator} objects (which contain data)
34       *         can be created.
35       */

37          return MediatorImpl.createEmpty();

45       *                supplied data.
46       * @param data the data the {@code Processor} created from this input {@code Mediator}.
47       * @return an output {@code Mediator} containing 'data' created by 'creator' using this
48       *         {@code Mediator} as input.
49       */
50      public abstract Mediator createNext(Processor creator, Object data);

52      /**
53       * Create a {@code Mediator} object after this one, which has been branched off and processed
54       * concurrently to give a list of mediators to join.  The list is saved in the new mediator's
55       * data.
56       * <p/>
57       * The returned object's {@code History} is the {@code History.join(..)} of this object's
58       * {@code History}, so if this method is ran twice with different {@code mediatorsToJoin} but
59       * which were created by the same {@code Processor} objects, the returned {@code Mediator}
60       * objects will have the same {@code History} object.  If they were produced by different
61       * {@code Processor} objects in the branches, then the returned {@code Mediator} objects
62       * would have different {@code History} objects.
63       * <p/>
64       * Calling {@code getPrevious()} on the returned {@code Mediator} object yields this object.
65       * The branched/joined {@code Mediator} objects are only accessible through the returned
66       * object's data field.
67       *
68       * @param creator the {@link Processor} that is joining the mediator objects.
69       * @param mediatorsToJoin the mediator objects to join.
70       * @return the next mediator object after the join.
71       */
72      public abstract Mediator join(Processor creator, List<Mediator> mediatorsToJoin);

110     /**
111      * For every {@code Mediator m} in {@code mediators}, this returns {@code m.getPrevious()} in
112      * a set.
113      * <p/>
114      * NB. Since duplicates are not allowed in a set, if any {@code Mediator} objects in
115      * {@code mediators} share the same previous {@code Mediator}, it will appear only once in
116      * the returned set.
117      *
118      * @param mediators the {@code Mediator} objects to roll back to their previous

121      */
122     public static Set<Mediator> rollbackMediators(Set<Mediator> mediators) {
123         Set<Mediator> previousMediators = new HashSet<>();
124         for (Mediator mediator : mediators)
