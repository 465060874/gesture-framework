8   /**
9    * A workflow is a linear list of {@link Element} objects which sits in a
10   * {@link WorkflowContainer}.
11   * <p/>
12   * When a workflow processes an input, it has its {@code Element} objects process the data
13   * sequentially, for example:
14   * {@code input -> lement1 -> data1 -> Element2 -> data2 -> ... -> LastElement -> output}.
15   * <p/>
16   * The {@code Workflow's} input and output types are independent of its {@code Elements},
17   * and are set at the {@code workflow's} construction.
18   * <p/>
19   * It is perfectly legal to have the wrong input/output types for the {@code workflow},
20   * or to have neighbouring elements be type-incompatible. In these situations,
21   * the workflow will return {@code false} for {@code isValid()}, and should have a way to inform
22   * the user (eg. in the associated view there might be connectors between neighbouring elements
23   * which are red if incompatible).
24   * <p/>
25   * NB. For two elements to be type-compatible, the previous {@code Element's} output type must be
26   * castable to the next {@code Element's} input type.
27   * <p/>
28   * However, a {@code Workflow} MUST be type-compatible with its parent {@code WorkflowContainer}.
29   * This is done so that all code related to checking neighbouring elements' data types is in
30   * {@code Workflow} (and nowhere else).  Otherwise all {@code WorkflowContainers} would have to
31   * worry about whether its {@code Workflows} match it's data types.  If this assertion is ever
32   * broken, a {@code ClassCastException} is thrown.
33   */
34  public interface Workflow extends Processor, ChildOf<WorkflowContainer>, ParentOf<Element> {

36      /**
37       * Returns the subset of this workflow's children which are invalid because their output type
38       * doesn't match the next element's input type.  If this workflow's input type doesn't match
39       * the first element's input type, the list will also contain a {@code null}.
40       *
41       * @return the child elements that are invalid due to their order in this workflow.
42       */
43      List<Element> getInvalidlyOrderedElements();
